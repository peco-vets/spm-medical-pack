# エンタープライズエージェント運用

単一 CLI セッションを超える運用統制が必要な、クラウドホスト型または継続稼働型のエージェントシステムに対して用いる。

## 運用ドメイン

1. ランタイムライフサイクル（start・pause・stop・restart）
2. 監視可能性（logs・metrics・traces）
3. 安全統制（scopes・permissions・kill switches）
4. 変更管理（rollout・rollback・audit）

## ベースライン統制

- イミュータブルなデプロイアーティファクト
- 最小権限クレデンシャル
- 環境レベルのシークレット注入
- ハードタイムアウトとリトライ予算
- 高リスクアクションの監査ログ

## 追跡すべきメトリクス

- 成功率
- タスクあたり平均リトライ回数
- 復旧時間
- 成功タスクあたりコスト
- 障害クラス分布

## インシデントパターン

障害が急増したとき:
1. 新規ロールアウトを凍結する
2. 代表トレースをキャプチャする
3. 障害ルートを隔離する
4. 最小の安全変更でパッチを当てる
5. 回帰とセキュリティチェックを実行する
6. 段階的に再開する

## デプロイ統合

このスキルは以下と組み合わせる。
- PM2 ワークフロー
- systemd サービス
- コンテナオーケストレータ
- CI/CD ゲート
