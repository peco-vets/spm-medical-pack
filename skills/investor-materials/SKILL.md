# 投資家資料

互いに整合し、信用でき、防御しやすい投資家向け資料を構築する。

## 起動タイミング

- ピッチデックの作成または改訂
- 投資家メモまたはワンページャーの記述
- 財務モデル・マイルストーン計画・use-of-funds 表の構築
- アクセラレーター/インキュベーター応募質問への回答
- 単一の真実源を中心に複数の資金調達ドキュメントを整列

## ゴールデンルール

すべての投資家資料は互いに合致しなければならない。

記述前に単一の真実源を作成または確認する:
- トラクションメトリクス
- 価格・売上仮定
- 調達規模と instrument
- 資金使途
- チームバイオとタイトル
- マイルストーンとタイムライン

矛盾する数字が現れたら、ドラフト前に止めて解決する。

## 中核ワークフロー

1. 正規事実をインベントリ化する
2. 欠落仮定を特定する
3. アセットタイプを選ぶ
4. 明示ロジックでアセットをドラフトする
5. すべての数字を真実源に対してクロスチェックする

## アセットガイダンス

### ピッチデック

推奨フロー:
1. 会社 + wedge
2. 問題
3. ソリューション
4. プロダクト / デモ
5. 市場
6. ビジネスモデル
7. トラクション
8. チーム
9. 競合 / 差別化
10. ask
11. 資金使途 / マイルストーン
12. 付録

ユーザーが Web ネイティブデックを希望する場合、`frontend-slides` と組み合わせる。

### ワンページャー / メモ

- 会社が何をするかを1つのクリーンな文で述べる
- "Why now" を示す
- トラクションと proof point を早期に含める
- ask を厳密にする
- 主張を検証しやすく保つ

### 財務モデル

以下を含める:
- 明示的仮定
- 有用な場合 bear / base / bull ケース
- 層別のクリーンな収益ロジック
- マイルストーン連動支出
- 仮定に意思決定が依存する場合の感度分析

### アクセラレーター応募

- 問われた厳密な質問に答える
- トラクション・洞察・チーム優位を優先する
- 誇張を避ける
- 内部メトリクスをデックとモデルと整合させる

## 避けるべきレッドフラグ

- 検証不能な主張
- 仮定なしのファジィな市場規模
- 矛盾するチーム役割やタイトル
- きれいに合計されない収益計算
- 仮定が脆弱な箇所での誇大な確実性

## 品質ゲート

引き渡し前に確認する:
- すべての数字が現真実源と一致する
- 資金使途と収益層が正しく合計される
- 仮定が埋没ではなく可視である
- ストーリーがハイプ言語なしに明確である
- 最終アセットがパートナーミーティングで防御可能である
