# 医療システム事業継続性（BCP）設計（SPM）

医療サービスは「停止が許されない」事業。SPM の各 Layer に求められる BCP 要件を設計レベルで整理。

## 医療 BCP の特殊性

通常システムの BCP との違い：
- **生命に直結** → ダウンタイムが直接的健康被害
- **法定義務**: 災害拠点病院は BCP 策定必須（医療法）
- **紙運用への切替が現実的**: 完全代替できる手書きフローが存在
- **データ復旧の優先順位**: 進行中の処置・処方 > 過去データ

## RPO/RTO 目標（SPM 推奨値）

| Layer | システム | RPO（許容データ損失） | RTO（許容停止時間） | 紙運用代替 |
|---|---|---|---|---|
| 1 | 院内システム（カルテ） | 5分以内 | 30分以内 | 可（カルテ用紙） |
| 1 | PECO アプリ | 1時間 | 6時間 | アプリは停止許容 |
| 2 | 診断 AI | 24時間（モデル） | 6時間 | 獣医師の判断で代替 |
| 3 | セントラル（全院連動） | 5分以内 | 1時間 | 拠点単独稼働で代替 |
| 4 | AI 調剤ロボット | 0（製造記録） | 即時 | 手動調剤に切替 |
| 5 | ゲノム/mRNA 製造 | 0（製造記録） | 製造ロット影響なき範囲 | 紙記録併用 |

## 必須実装要件

### 1. データ冗長化
- [ ] DB レプリケーション（同期 or 準同期）
- [ ] 地理的に離れた拠点へのバックアップ（東京 + 大阪 + クラウド）
- [ ] バックアップの定期復元テスト（最低四半期に1回）
- [ ] WAL アーカイブ等で PITR（Point-in-Time Recovery）可能に

### 2. 拠点間連動の障害耐性（Layer3）
- [ ] センター病院（原宿）の機能を各拠点が部分代替可能
- [ ] 拠点間通信断時も拠点内で完全な診療業務継続可能
- [ ] ネットワーク復旧後の自動同期（コンフリクト解決ロジック）
- [ ] 「最終的整合性」前提の設計

### 3. アプリケーション層の障害耐性
- [ ] サーキットブレーカー（外部 API 障害時の遮断）
- [ ] グレースフルデグラデーション（一部機能停止でも主機能継続）
- [ ] リトライ＋指数バックオフ（外部依存呼出）
- [ ] タイムアウト設定（無限待ち禁止）

### 4. 監視・検知
- [ ] ヘルスチェック（アプリ・DB・外部依存）
- [ ] アラート閾値設定（CPU・メモリ・レスポンスタイム・エラー率）
- [ ] オンコール体制
- [ ] インシデント対応手順書

### 5. 紙運用切替の組込
- [ ] システム停止検知時、各院に紙運用切替を即時通知
- [ ] 紙運用用テンプレート（カルテ用紙・処方箋）のオフライン保管
- [ ] システム復旧時の紙データ取込フロー
- [ ] 切替訓練の年次実施

実装テンプレ：
```typescript
// 各 API ハンドラに組込
async function withMedicalBcp<T>(
  operation: () => Promise<T>,
  fallback: { 
    paperProcedure: string;  // 紙運用手順 ID
    severity: "minor" | "major" | "critical";
  }
): Promise<T> {
  try {
    return await Promise.race([
      operation(),
      timeoutPromise(5000),
    ]);
  } catch (e) {
    await notifyOnCall({ error: e, severity: fallback.severity });
    if (fallback.severity === "critical") {
      await broadcastPaperFallback(fallback.paperProcedure);
    }
    throw new MedicalServiceUnavailable(fallback);
  }
}
```

### 6. ディザスタリカバリ
- [ ] DR 訓練の年次実施
- [ ] 復旧手順書（Runbook）の整備
- [ ] 主要拠点全断時の対応プラン
- [ ] サードパーティ依存（クラウド・SaaS）の障害時対応

## 法的な事業継続義務

- 医療法施行規則: 災害拠点病院は BCP 策定義務
- 厚労省「医療機関等における BCP の概要等」参照
- 動物病院は法定義務なしだが、150院展開時には実質必須

## 設計時のチェック観点

1. [ ] この機能が停止したら患者／飼主の生命・安全に影響あるか？
2. [ ] RPO/RTO 目標値は決まっているか？
3. [ ] 紙運用代替フローは定義されているか？
4. [ ] 障害検知から通知までの時間目標は？
5. [ ] DB レプリケーションは正しく設定されているか？
6. [ ] 復旧テストは実施されているか？

## 違反時のリスク

- 患者／飼主への直接的健康被害 → 損害賠償・訴訟
- 法的事業継続義務違反（災害拠点病院相当の場合）
- レピュテーション毀損・契約解除
