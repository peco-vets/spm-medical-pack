# SaMD（プログラム医療機器）該当性判定フロー（SPM）

Layer2 診断 AI 等の機能が薬機法の「プログラム医療機器」に該当するかを判定し、該当時の規制対応を整理。**事業継続性に直結する重大論点**。

## SaMD とは

薬機法（医薬品医療機器等法）2条4項：
> 疾病の診断、治療、予防に使用されること又は身体の構造若しくは機能に影響を及ぼすことを目的とするプログラム

該当すると医療機器として認証／承認が必要。

## 該当性判定フロー（PMDA 通知に基づく）

### Step 1: 目的判定
- このプログラム／AI 機能の目的は？
  - 疾病の診断 → 該当の可能性
  - 治療方針の提示 → 該当の可能性
  - 治療効果の判定 → 該当の可能性
  - 単なる情報提供・教育 → 非該当
  - 業務効率化（受付・予約・在庫等） → 非該当

### Step 2: リスク判定
- プログラムの結果が患者に与える影響度は？
  - 重大（生命に関わる判定） → 該当濃厚、高クラス
  - 中程度（治療方針変更等） → 該当、中クラス
  - 軽微（参考情報程度） → 該当の場合でも低クラス

### Step 3: 自律性判定
- 医師／獣医師が必ず確認・判断するか？
  - AI が単独で確定診断 → 該当濃厚
  - AI は補助、医師が最終判断 → 該当する場合でも低クラス
  - 単なる情報表示 → 非該当の可能性

### Step 4: クラス分類（該当時）

| クラス | リスク | 例 | SPM での該当例 |
|---|---|---|---|
| クラスI | 一般医療機器（低） | 病院情報システム単体 | 受付システム単体 |
| クラスII | 管理医療機器（低中） | 画像診断支援（補助情報提示） | Layer2 補助診断機能 |
| クラスIII | 高度管理医療機器（高） | 確定診断機能 | Layer2 自律診断（将来） |
| クラスIV | 高度管理（最高） | 生命維持・侵襲的 | 該当想定なし |

### Step 5: 認証・承認ルート

- **クラスI**: 製造販売届出（届出のみ、PMDA 審査なし）
- **クラスII**: 第三者認証機関による認証（QMS 適合性調査含む）
- **クラスIII**: PMDA 承認 + QMS + 臨床試験
- **クラスIV**: PMDA 承認 + QMS + 大規模臨床試験

## 動物用医療機器の特殊性

動物用医療機器も薬機法適用（83条で準用）：
- 規制主体：農林水産省（人医療は厚労省）
- 動物用医療機器の承認・認証は農水省 / 動物医薬品検査所
- 獣医療 AI も対象になり得る

## SPM Layer2 の現状判定（仮）

> 注：以下は技術的論点整理であり、最終判定は弁護士／規制コンサル必須

- **症状チェッカー**（一般情報提供レベル）: 非該当の可能性
- **画像診断補助**（獣医師が必ず判断）: クラスII該当可能
- **確定診断 AI**（自律判定）: クラスIII該当濃厚

## 該当時の開発体制要件（QMS Reg）

- ISO 13485 準拠の品質マネジメントシステム
- 設計管理（DHF: Design History File）
- リスクマネジメント（ISO 14971）
- ソフトウェアライフサイクル（IEC 62304）
- 市販後監視（PMS）

## 設計時の判断ポイント

機能追加・変更時に毎回チェック：

1. [ ] この変更は SaMD 該当性に影響するか？
2. [ ] 既に該当している場合、クラス分類が変わるか？
3. [ ] 変更後の臨床的影響は？
4. [ ] 大幅な変更なら一部変更承認申請が必要か？

## 該当を避ける設計パターン

完全回避は難しいが、低クラスに留めるパターン：
- AI の出力は必ず「参考情報」「補助情報」として提示
- 医師／獣医師の確認・承認なしに次のアクション不可
- 確定的な「診断」表現を避け、「○○の可能性」表現に
- 治療方針は医師が選択肢から選ぶ UI

## 関連法規

- 薬機法（医薬品、医療機器等の品質、有効性及び安全性の確保等に関する法律）
- 厚労省「プログラムの医療機器該当性に関するガイドライン」
- PMDA「次世代医療機器・再生医療等製品評価指標策定事業」

## 違反時のリスク

- 無承認医療機器の販売 → 業務停止・刑事罰
- Layer2 の事業展開不能リスク
- リコール・損害賠償
