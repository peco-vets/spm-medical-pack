# 要配慮個人情報の取扱フロー（SPM／APPI 改正対応）

個人情報保護法（APPI）改正版に基づく、医療情報（＝要配慮個人情報）の取扱いを実装レベルで満たすフロー集。

## 要配慮個人情報の定義

以下を含む情報は「要配慮個人情報」（APPI 2条3項）：
- 病歴（既往歴・治療歴・服薬歴）
- 健康診断・検診結果
- 医師等による診療・調剤
- 身体障害・知的障害・精神障害
- 遺伝情報・DNA 型
- 犯罪歴・犯罪被害情報
- 人種・信条・社会的身分

→ SPM は全 Layer で要配慮個人情報を扱う

## 通常の個人情報との違い

| 項目 | 個人情報 | 要配慮個人情報 |
|---|---|---|
| 取得時の同意 | 利用目的の通知/公表で OK | **本人/飼主の明示的同意必須** |
| オプトアウト第三者提供 | 可 | **不可（明示同意必須）** |
| 漏えい時の報告 | 1000人超 or 不正アクセス時 | **件数問わず必須** |

## 設計フロー（コード書く前のチェック）

### Phase 1: 適用判定
1. このコードは何の情報を扱う？ → 病名 / 症状 / 遺伝情報 等が含まれる？
2. 含まれるなら → 以下のチェック全て必須

### Phase 2: 取得（カルテ作成・問診票等）
- [ ] 同意取得 UI／フローが組み込まれているか
- [ ] 同意取得の記録（誰が・いつ・何の目的で）を DB に保持
- [ ] 利用目的の通知文を表示
- [ ] 撤回手段の明示

実装テンプレ：
```typescript
model ConsentRecord {
  id           String   @id @default(cuid())
  patientId    String   // または ownerId（飼主）
  purposes     String[] // ["診療", "AI学習", "ゲノム研究"] 等
  agreedAt     DateTime
  agreedMethod String   // "tablet_signature" | "verbal_witnessed" | "web_form"
  witnessedBy  String?  // 受付スタッフ ID
  revokedAt    DateTime?
  documentVersion String // 同意書のバージョン
}
```

### Phase 3: 利用
- [ ] 取得時の目的範囲内での利用か（目的外利用は別途同意必要）
- [ ] 最小限の項目のみ取得・利用
- [ ] AI 学習に使う場合は同意項目に明示

### Phase 4: 保存
- [ ] 暗号化（保存時：DB レベル AES-256 推奨）
- [ ] アクセス制御（RBAC）
- [ ] アクセスログ記録
- [ ] バックアップ／復旧（[[spm-electronic-record-3principles]] と連携）

### Phase 5: 第三者提供（病院間連携・研究機関提供等）
- [ ] **要配慮個人情報のオプトアウト第三者提供は禁止** — 必ず明示同意
- [ ] 提供先・提供項目・提供方法を同意書に明記
- [ ] 提供記録を保管（提供日時・先・項目）

### Phase 6: 委託（クラウド事業者・分析業者等）
- [ ] 委託契約に安全管理措置を明記
- [ ] 委託先の監督（年次監査等）
- [ ] サブ委託は事前承諾制が望ましい

### Phase 7: 外国移転
- [ ] 海外サーバ（米国 AWS 等）への保存は「外国にある第三者への提供」に該当する場合あり
- [ ] EU 圏は GDPR、米国は EU-US DPF 等の枠組み確認
- [ ] 同意取得時に移転先国を明示

### Phase 8: 匿名化・仮名化（研究・統計利用）

| 種別 | 復元可能性 | 利用範囲 | 法的扱い |
|---|---|---|---|
| 仮名加工情報 | 識別子の対応表があれば復元可 | 内部統計分析等 | 個人情報のまま、第三者提供制限あり |
| 匿名加工情報 | 復元不可（k-匿名性等で担保） | 第三者提供可・公表可 | 個人情報外 |

匿名加工する場合の最低要件：
- [ ] 特定の個人を識別できる記述等の削除（氏名・ID・電話番号等）
- [ ] 個人識別符号の全部削除
- [ ] 加工方法の記録・保管
- [ ] 公表（取扱う旨・項目）

### Phase 9: 漏えい時対応（必ず実装する報告フロー）

要配慮個人情報の漏えいは **件数問わず** 個人情報保護委員会への報告必須：
- [ ] 漏えい検知の仕組み（異常アクセス検知・脆弱性アラート）
- [ ] 検知から **3〜5日以内** に速報 → **30日以内** に確報
- [ ] 本人通知のフロー
- [ ] インシデント対応手順書

## 動物医療特有の論点

獣医療では飼主が「個人情報の主体」、動物自体は「物」扱い：
- 飼主氏名・住所・連絡先 → 個人情報
- 動物の病歴 → 飼主が特定される場合、要配慮個人情報に該当する可能性
- 飼主同意が原則
- ただし「ペット保険」連携時等は飼主の明示同意必須

## 違反時のリスク

- 個人情報保護委員会の改善命令／勧告
- 1億円以下の罰金（法人の場合）
- 損害賠償・レピュテーション
