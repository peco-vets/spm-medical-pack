# Connections Optimizer

アウトバウンドを一方向のプロスペクティングリストとして扱うのではなく、ユーザーのネットワークを再編成する。

このスキルは以下を扱う:

- X フォローのクリーンアップと拡張
- LinkedIn フォローとコネクション分析
- レビューファーストのプルーニングキュー
- 追加とフォローの推奨
- ウォームパス特定
- ユーザーの実ボイスでの Apple Mail・X DM・LinkedIn ドラフト生成

## 起動するタイミング

- ユーザーが X フォローをプルーニングしたい
- ユーザーがフォローや接続を保つ相手を再バランスしたい
- ユーザーが「ネットワークをクリーンアップ」「誰をアンフォローすべき」「誰をフォローすべき」「誰と再接続すべき」と言う
- アウトリーチ品質が単なるコールドリスト生成ではなくネットワーク構造に依存する

## 必要な入力

収集または推測する:

- 現在の優先順位とアクティブな仕事
- ターゲットの役割、業界、地理、エコシステム
- プラットフォーム選択: X、LinkedIn、または両方
- do-not-touch リスト
- モード: `light-pass`・`default`・`aggressive`

ユーザーがモードを指定しない場合、`default` を使う。

## ツール要件

### 優先

- X グラフ検査と最近のアクティビティのための `x-api`
- ターゲット発見とウォームパスランキングのための `lead-intelligence`
- 広範なリードワークフローとは独立にブリッジ値をスコアしたいときの `social-graph-ranker`
- 人と会社のエンリッチメントのための Exa / deep research
- アウトバウンドドラフト前の `brand-voice`

### フォールバック

- LinkedIn 分析とドラフトのためのブラウザ制御
- API カバレッジが制約されているときの X 用ブラウザ制御
- メールが正しいチャネルのときのデスクトップ自動化による Apple Mail または Mail.app ドラフト

## 安全デフォルト

- デフォルトはレビューファースト、決してブラインドな自動プルーニングではない
- X: ユーザーがフォローしているアカウントのみプルーン、決してフォロワーをプルーンしない
- LinkedIn: 1 親等接続削除を手動レビューファーストとして扱う
- DM・招待・メールを自動送信しない
- 任意の apply ステップ前にランク付きアクションプランとドラフトを発行する

## プラットフォームルール

### X

- 相互フォローは一方向フォローよりも粘着性が高い
- ノンフォローバックはより積極的にプルーンできる
- 大幅に非アクティブまたは消えたアカウントは素早く浮上させる
- エンゲージメント、シグナル品質、ブリッジ値は生のフォロワー数より重要

### LinkedIn

- ユーザーが実際に LinkedIn API アクセスを持つなら API ファースト
- API アクセスが欠落しているときブラウザワークフローが機能しなければならない
- アウトバウンドフォローと受諾された 1 親等接続を区別する
- アウトバウンドフォローはより自由にプルーンできる
- 受諾された 1 親等接続は自動削除ではなくデフォルトでレビューすべき

## モード

### `light-pass`

- 高信頼度の低価値一方向フォローのみプルーン
- 残りをレビュー用に表面化
- 小さな追加/フォローリストを生成

### `default`

- バランス取れたプルーニングキュー
- バランス取れた keep リスト
- ランク付き追加/フォローキュー
- 有用な場所でウォームイントロまたは直接アウトリーチをドラフト

### `aggressive`

- 大きめのプルーニングキュー
- 古いノンフォローバックへの低い許容度
- 依然として apply 前にレビューゲート

## スコアリングモデル

これらのポジティブシグナルを使う:

- 相互性
- 最近のアクティビティ
- 現在の優先順位への整合性
- ネットワークブリッジ値
- 役割関連性
- 実エンゲージメント履歴
- 最近のプレゼンスと応答性

これらのネガティブシグナルを使う:

- 消えたまたは放棄されたアカウント
- 古い一方向フォロー
- 優先順位外のトピッククラスタ
- 低価値ノイズ
- 繰り返される非応答
- より良い置き換えが多くあるときのフォローバックなし

相互フォローと実ウォームパスブリッジは一方向フォローよりも積極的にペナルティを下げるべき。

## ワークフロー

1. 優先順位、do-not-touch 制約、選択プラットフォームをキャプチャ。
2. 現在のフォロー / コネクションインベントリを取得。
3. 明示的な理由付きでプルーン候補をスコア。
4. 明示的な理由付きで keep 候補をスコア。
5. 拡張候補をランクするために `lead-intelligence` プラスリサーチサーフェスを使う。
6. 正しいチャネルをマッチ:
   - ウォーム・素早いソーシャルタッチポイントには X DM
   - プロフェッショナルグラフ隣接には LinkedIn メッセージ
   - 高コンテキストイントロやアウトリーチには Apple Mail ドラフト
7. メッセージドラフト前に `brand-voice` を実行。
8. 任意の apply ステップ前にレビューパックを返す。

## アウトバウンドルール

- デフォルトのメールパスは Apple Mail / Mail.app ドラフト作成。
- 自動送信しない。
- 温かさ、関連性、コンテキスト深度に基づいてチャネルを選ぶ。
- メールやアウトリーチなしが正しい動きのとき DM を強制しない。
- ドラフトは自動セールスコピーではなくユーザーのように聞こえるべき。

## 関連スキル

- 再利用可能ボイスプロファイルのための `brand-voice`
- スタンドアロンブリッジスコアリングとウォームパス計算のための `social-graph-ranker`
- 重み付きターゲットとウォームパス発見のための `lead-intelligence`
- X グラフアクセス、ドラフト、オプション適用フローのための `x-api`
- ユーザーがネットワーク移動の周辺でパブリックローンチコンテンツも望むときの `content-engine`
