# 3省2ガイドライン第6.0版 準拠チェック（SPM）

医療情報システムの安全管理ガイドライン（3省2GL）への準拠状況を、コード・設計レベルで検証するための実用チェックリスト。

## 適用判定（最初に判定）

以下のいずれかに該当 → 本ガイドライン適用：
- 診療録・カルテ・処方・検査結果・画像データを保存／処理する
- 患者・飼主の個人情報を取り扱う（氏名 + 病歴等の組み合わせ）
- 院内システム同士／センター病院（原宿）と拠点病院の間で医療情報を送受信する
- クラウドサービス上で医療情報を扱う（厚労省 GL + 経産省・総務省 GL の両方適用）

## 4つの安全管理対策カテゴリ

### 1. 組織的安全管理対策
- [ ] 医療情報システム安全管理責任者の指定（コード上は不要だが、運用フローに明記）
- [ ] 医療情報の取扱規程・運用マニュアルへの参照リンクが README にあるか
- [ ] アクセス記録の定期点検手順が運用ドキュメント化されているか

### 2. 物理的安全管理対策
- [ ] サーバ設置場所のアクセス制限（クラウド側は事業者責任）
- [ ] バックアップ媒体の物理保護（クラウド冗長化で代替可）
- [ ] 端末（iPad 等）の物理的盗難・紛失対策（リモートワイプ機能）

### 3. 技術的安全管理対策（コードレビュー必須項目）

#### 認証
- [ ] パスワード／生体認証／IC カード等で本人識別（最低2要素を推奨）
- [ ] パスワード強度ポリシー（長さ・複雑性）
- [ ] アカウントロックアウト（連続失敗 N 回で一時停止）
- [ ] セッションタイムアウト（一定時間無操作で自動ログアウト）

#### アクセス制御
- [ ] ロールベースアクセス制御（RBAC）— 院長・獣医師・看護師・受付で権限分離
- [ ] 最小権限の原則（必要な情報のみ参照可）
- [ ] 他拠点の診療情報参照には明示的な権限フロー

#### 暗号化
- [ ] 保存時暗号化（DB レベル or アプリレベル）
- [ ] 通信暗号化（TLS 1.2 以上、HSTS 設定）
- [ ] 暗号鍵管理（KMS 利用、ハードコード禁止）

#### 監査ログ（重要）
- [ ] 「誰が・いつ・何を・どうしたか」を全アクセス記録
- [ ] ログの改ざん防止（追記専用ストレージ、ハッシュチェーン）
- [ ] ログ保存期間（少なくとも 3年、推奨 5年以上）
- [ ] ログの定期点検フロー

#### 不正アクセス対策
- [ ] WAF / IDS / 異常検知の設定
- [ ] SQL インジェクション / XSS / SSRF / CSRF 対策（OWASP Top 10 全件）
- [ ] 依存ライブラリの脆弱性スキャン（npm audit / Snyk 等）の CI 組込

#### バックアップ・障害対策
- [ ] 定期バックアップ（最低日次、世代管理）
- [ ] 復旧手順の文書化と訓練
- [ ] 災害時の業務継続性（紙運用への切替手順）

### 4. 人的安全管理対策
- [ ] アクセス権限の入退社管理フロー
- [ ] 医療情報取扱の教育記録

## SaaS/クラウド提供者としての追加要件（経産省・総務省 GL）

SPM 自体がクラウドサービス提供者となる場合：
- [ ] 利用者（医療機関）との責任分界点の明示（SLA / 利用規約）
- [ ] サブ委託先の管理（IaaS/PaaS 事業者の選定基準）
- [ ] 提供サービスの監査受入体制
- [ ] サービス終了時のデータ返却・消去手順

## チェック実行コマンド（参考）

```bash
# OWASP 脆弱性スキャン
npx audit-ci --moderate

# シークレットスキャン
git diff --cached | grep -iE "password|secret|api_key|token"

# 暗号化設定確認（Prisma 例）
grep -r "encryption" prisma/schema.prisma

# 監査ログ存在確認
grep -r "auditLog\|accessLog" src/
```

## 違反時のリスク

- 医療法・個人情報保護法違反 → 行政指導／改善命令／業務停止命令
- 漏えい時 → 個人情報保護委員会への 72時間以内報告義務 + 本人通知
- 損害賠償・レピュテーション毀損
