# Social Graph Ranker

ネットワーク認識アウトリーチ向けの正規重み付きグラフランキング層。

ユーザーが以下を必要とするときに使う：

- 既存の相互フォローまたはコネクションを紹介価値でランク付け
- ターゲットリストへの温かいパスをマップ
- 一次・二次コネクションにわたるブリッジ価値を測定
- 直接的なコールドアウトリーチではなく温かい紹介に値するターゲットを決定
- `lead-intelligence` や `connections-optimizer` から独立してグラフ数学を理解

## このスキルを単独で使うタイミング

ユーザーが主にランキングエンジンを求めるときにこのスキルを選ぶ：

- 「自分のネットワークの中で、私を紹介するのに最適な位置にいるのは誰か？」
- 「これらの人々に私を取り次げる人で相互フォローをランク付けして」
- 「私のグラフをこの ICP に対してマップして」
- 「ブリッジ数学を見せて」

ユーザーが本当に以下を求めているとき、これを単独で使わない：

- 完全なリードジェネレーションとアウトバウンドシーケンス → `lead-intelligence` を使う
- ネットワークの剪定、リバランシング、成長 → `connections-optimizer` を使う

## 入力

収集または推論する：

- ターゲット人物、企業、または ICP 定義
- ユーザーの X、LinkedIn、または両方の現在のグラフ
- 役割、業界、地理、応答性などの重み付け優先度
- トラバーサル深度と減衰許容度

## コアモデル

与えられた：

- `T` = 重み付きターゲットセット
- `M` = あなたの現在の相互フォロー／直接コネクション
- `d(m, t)` = 相互フォロー `m` からターゲット `t` への最短ホップ距離
- `w(t)` = シグナルスコアリングからのターゲット重み

基本ブリッジスコア：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
```

ここで：

- `λ` は減衰係数、通常 `0.5`
- 直接パスはフル価値を貢献
- 各追加ホップは貢献を半減

二次拡張：

```text
B_ext(m) = B(m) + α · Σ_{m' ∈ N(m) \\ M} Σ_{t ∈ T} w(t) · λ^(d(m',t))
```

ここで：

- `N(m) \\ M` は相互フォローが知っていてあなたが知らない人々のセット
- `α` は二次到達を割り引く、通常 `0.3`

応答調整済み最終ランキング：

```text
R(m) = B_ext(m) · (1 + β · engagement(m))
```

ここで：

- `engagement(m)` は正規化された応答性または関係強度
- `β` はエンゲージメントボーナス、通常 `0.2`

解釈：

- ティア 1：高 `R(m)` と直接ブリッジパス → 温かい紹介依頼
- ティア 2：中 `R(m)` と 1 ホップブリッジパス → 条件付き紹介依頼
- ティア 3：低 `R(m)` または実行可能なブリッジなし → 直接アウトリーチまたはフォローギャップ補充

## スコアリングシグナル

現在の優先セットにとって重要なものでグラフトラバーサル前にターゲットを重み付ける：

- 役割または肩書きの整合性
- 企業または業界フィット
- 現在のアクティビティと最新性
- 地理的関連性
- 影響力または到達
- 応答可能性

トラバーサル後に相互フォローを以下で重み付ける：

- ターゲットセットへの重み付きパスの数
- それらのパスの直接性
- 応答性または先行インタラクション履歴
- 紹介を行うためのコンテキストフィット

## ワークフロー

1. 重み付きターゲットセットを構築する
2. X、LinkedIn、または両方からユーザーのグラフを引く
3. 直接ブリッジスコアを計算する
4. 最高価値の相互フォローについて二次候補を拡張する
5. `R(m)` でランク付けする
6. 返す：
   - 最良の温かい紹介依頼
   - 条件付きブリッジパス
   - 温かいパスが存在しないグラフギャップ

## 関連スキル

- `lead-intelligence` は、より広範なターゲット発見とアウトリーチパイプラインの中でこのランキングモデルを使う
- `connections-optimizer` は、誰を維持、剪定、追加するかを決める際に同じブリッジロジックを使う
- `brand-voice` は紹介依頼または直接アウトリーチを起草する前に実行すべき
- `x-api` は X グラフアクセスとオプションの実行パスを提供する
