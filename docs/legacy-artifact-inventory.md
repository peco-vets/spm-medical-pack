# レガシーアーティファクトインベントリ

本インベントリはレガシーと陳腐作業クリーンアップが暗黙的にならないようにする。各アーティファクトはリリース作業がキューをクリーンと扱う前に landed、マイルストーン追跡、サルベージブランチ、archive/no-action として分類されるべきである。

## 分類状態

| 状態 | 意味 |
| --- | --- |
| Landed | 有用な作業が現 `main` に既に port され検証済み。 |
| マイルストーン追跡 | 有用な作業が残るが、named ロードマップマイルストーンに属する。 |
| サルベージブランチ | 有用な作業は帰属付きで新メンテナブランチを通じて port されるべき。 |
| 翻訳者/手動レビュー | コンテンツは有用かもしれないが、自動的に安全にインポートできない。 |
| Archive/no-action | アーティファクトは意図的に保持またはスキップ。アクティブな port は計画されていない。 |

## ワークスペースレガシーインポートルール

ワークスペースレベルレガシーリポジトリをマイニングする際:

1. `.env` ファイル、トークン、OAuth シークレット、ローカル設定、個人パス、プライベートオペレータコンテキストを読み、出力、ステージ、コピーしない。
2. 生マーケティングドラフト、game プラン、チャット/コンテキストダンプをインポートしない。
3. フォーカスされた公開安全アイデアのみを現ドキュメントまたはコードに抽出する。
4. 新 PR でソースレガシーリポジトリ、ブランチ、コミット、または陳腐 PR を帰属する。
5. ネイティブ作業と同じテストとリリースチェックで結果を検証する。

## リリースルール

任意の GA または rc 公開パス前に:

1. `_legacy-documents-*` スキャンを再実行する。
2. closed-stale サルベージ台帳チェックを再実行する。
3. 新発見された各レガシーアーティファクトが本ファイルに表現されることを確認する。
4. ソース帰属付きで新メンテナ PR を通じて有用な作業を port する。
5. archive/no-action アーティファクトをデフォルトインストールとプラグインロードから外す。
