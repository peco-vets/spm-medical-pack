## 所見

### Medium

1. `skills/continuous-learning-v2/hooks/observe.sh`

新しい `CLAUDE_CODE_ENTRYPOINT` ガードは既知の非 `cli` 値 (`sdk-ts`、`sdk-py`、`sdk-cli`、`mcp`、`remote`) の有限な allowlist を使う。

これは前方互換性の穴を残す: 将来の任意の非 `cli` エントリポイント値はフォールスルーし、対話型として扱われる。これは PR が防ごうとしているまさにそのクラスの自動化セッション観測を再導入する。

より安全なルールは:

- `cli` のみ許可
- 他の明示的エントリポイントはすべて自動化として扱う
- 変数が unset のときデフォルトフォールバックを `cli` とする

推奨形状:

```bash
case "${CLAUDE_CODE_ENTRYPOINT:-cli}" in
  cli) ;;
  *) exit 0 ;;
esac
```
